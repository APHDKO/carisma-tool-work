UMLsec Static Fair Exchange Check

Overview

This check analyzes UML activity diagrams which uses the UMLsec stereotype «fair exchange» (cf. "Secure Systems Development with UML", p. 53 ff.). For the state marked as «start», the check analyzes if there occurs a state marked as «stop» after it in any possible paths.

Annotate a package containing a state machine with «fair exchange». Apply the appropriate «start» and «stop» tags in the corresponding states.

Afterwards create a new CARiSMA analysis on the model. Add the Static Fair Exchange Check to the used checks and click "Run" to start the analysis. The Analysis Results View displays whether the check was successful. If any possible path didn't fit the test, the path will be displayed in the Analysis Report.
